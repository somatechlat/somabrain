## Review Process

1. **Author** creates or updates documentation
2. **Technical Reviewer** verifies accuracy against code
3. **Documentation Lead** checks style and completeness
4. **Approver** signs off for publication

## Sign-off Requirements

- Technical accuracy verified against source code
- All code examples tested and working
- Configuration values match actual defaults
- API endpoints verified against `somabrain/app.py`
- No placeholder or invented content
